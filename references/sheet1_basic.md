# 1枚目（基本情報）ヒアリングガイド（第2回公募版）

## ヒアリング順（推奨）

ユーザー負担を軽減するため、AskUserQuestion はテーマ別にまとめる。第2回公募ではAIインタビューが廃止されたため「メアド一致」要件は撤廃された（所属機関発行のメアド要件は継続）。

### Q1. 研究者情報（まとめて1つのメッセージで依頼）

- e-Rad 研究者番号（8桁の半角数字、学生応募で自身が持たない場合は指導教員の番号）→ `C8`
- 所属機関から発行されているメールアドレス → `C10`
- 氏名（フリガナ＋漢字、姓名の間は半角1スペース）→ `D12`, `D13`
  - 第2回公募の留意事項では「半角1スペース」とされている（第1回の「全角1スペース」から変更）
  - **ファイル名用の氏名はこれとは別に確認する**：ファイル名に使う氏名は **スペース・アンダースコアなしの連結形** が必要（例: `山田太郎` / `YamadaTaro`）。詳細はSKILL.md の Step 6「ファイル名の規則」参照
- 生年月日（西暦・半角）→ `C15`（年）, `F15`（月）, `H15`（日）
- e-Rad 所属機関コード（**10桁の半角数字**、先頭ゼロを含む場合あり、不明時は所属機関に確認）→ `C16`
  - 科研費の機関番号とは別物
- 所属機関／部局／職 → `C18`, `C19`, `C20`
- 学生フラグ：博士／修士／学部学生のいずれかなら `L20` に `Y`
- **所属機関の区分** → `C21`（`lists.md` の C 列：大学／高等専門学校／公的研究機関／民間企業／非営利団体・公益法人／その他）
- **応募者属性の区分** → `C23`（`lists.md` の D 列：教員／研究員(ポスドク含む)／博士／修士／学部／技術職員・URA等／技術者・開発者／その他）

学生応募で指導教員の研究者番号を使う場合も、氏名・生年月日は**学生本人**のものを記入する。応募者属性の区分も学生本人のものを選ぶ。

### Q1.5. 提出日

- 提出予定日 → `H6`（年）, `J6`（月）, `L6`（日）
- 機関等が指定する提出期限（公募締切は令和8年7月3日正午）

### Q2. 研究領域・ユースケース（AskUserQuestion）

- 研究領域 → `C27`（11区分から1つ、`lists.md` 参照）
- メインユースケース → `C29`（9区分から1つ、「9.その他」は最後の手段）
- 「9.その他」を選んだ場合 → `C31` に自由記述（必須）
- サブユースケース → C32, E32, G32, I32, C33, E33, G33, I33 のうち該当する欄に `Y`

### Q3. 研究課題名

- `C35`：日本語50字程度（目安、上限/下限のエラーは出ない）
- 英語版テンプレを使う場合は同じ `C35` に約30 words の英語題

### Q4. AI活用度（複数選択可・10項目）

`lists.md` の10項目から該当するものを選び、対応セルに `Y` を入れる：
- C39: AIをまったく使っていない
- E39: 研究以外の業務でAI
- G39: 文献探索・要約
- I39: 論文・発表資料作成
- K39: 仮説検討・アイデア出し
- C40: AIでデータ分析（自作コード含む）
- E40: AI分析結果を論文・発表で発表
- G40: API で既存AIを研究プロセスに組み込み
- I40: AIモデル開発の経験あり
- K40: 新しい基盤モデル開発の経験あり

### Q5. 現在の具体的な活用方法

- `C42`：日本語200字程度／英語120 words 程度（目安、エラー判定なし）

## 書き込み時の注意

- 第2回テンプレートでは **サブユースケース・AI活用度は "Y" を入れる方式**（第1回の boolean True/False 方式から変更）。プルダウンが効かない環境でもユーザーが手で `Y` を打てる仕様。
- `N35`（`=LEN(C35)`）と `N42`（`=LEN(C42)`）は既存数式を保持する。
- 「整理番号」「機関番号」は第2回ではテンプレ表面から削除されているので応募者入力不要。
- `H20:K20` は学生応募の説明テキスト（学生フラグの隣にある注釈）なので入力不要。
