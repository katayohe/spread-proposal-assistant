# 3枚目（研究経費の内訳）ガイド（第2回公募版）

## 大前提

- **直接経費の上限：1課題あたり 500 万円 以下**（**下限 10 万円以上**、10 万円未満の研究計画は対象外）
- **間接経費：直接経費の30%**（機関に配分、調書には計上しない）
- **人件費は本事業では対象外**
- **再委託費・共同実施費も対象外**
- **消費税相当額は補助事業のため対象外**
- **xlsx 様式の金額欄は「千円」単位で固定**されているため、書き込む時のみ `万円 × 10 = 千円` に換算。人が話す／メモする時は **万円単位** で統一する。
- 千円未満は切り捨て
- 費目間流用は**文科省の承認なしで直接経費総額の50%以内**
- 採択額は審査結果の充足率を 500 万円に乗じた基準配分額を上限とし、申請額を超えない範囲で決定される（申請額が下回る場合は原則として満額採択）

## 第2回公募で変わった点

- **3枚目は行追加禁止**。設備備品費／消耗品費／謝金／国内旅費／外国旅費はそれぞれ20行固定枠、その他費用も20行固定。
- 設備備品費の金額列がテンプレ式 `=G{row}*I{row}` で自動計算（単価×数量）。
- 各費目の小計セルもテンプレ式 `=SUM(...)` で自動計算。スクリプト側では明細行のみ書き込み、小計セルは触らない。

## 費目構成（府省共通経費取扱区分表）

### 直接経費

| 大項目 | 中項目 | 3枚目テンプレ位置 |
|---|---|---|
| 物品費 | 設備備品費 | 行11〜30、列 D事項 / E:F設置機関 / G:H単価 / I数量 / J金額(数式) |
| 物品費 | 消耗品費 | 行11〜30、列 M事項 / N金額 |
| 人件費・謝金 | 人件費 | **対象外** |
| 人件費・謝金 | 謝金 | 行40〜59、列 D事項 / E金額 |
| 旅費 | 国内旅費 | 行40〜59、列 H:I事項 / J金額 |
| 旅費 | 外国旅費 | 行40〜59、列 M事項 / N金額 |
| その他 | 外注費／印刷製本費／会議費／通信運搬費／光熱水料／その他（諸経費） | 行69〜88、列 D事項 / E金額 |

### 計上ルール（SPReAD 固有）

- **設備備品費**：GPU 搭載サーバ、ワークステーション、研究用ストレージ装置等で**備品として管理すべきもの**。計算資源を物品として取得する場合（GPUカード単独購入等）も設備備品費に計上し、4枚目に詳細を書く。
- **消耗品費**：安価な周辺機器、記録媒体、交換部材等で**消耗品として管理すべきもの**
- **AWS 等民間クラウド利用料（物品購入を伴わない）→ 4枚目「計算資源費用」テーブルへ、3枚目では「その他」の事項に計上**
- **API 利用料（従量課金 LLM 等）→ 4枚目「API費用」テーブルへ、3枚目では「その他」に計上**
- 「その他（諸経費）」として、英文校正、論文掲載料、研究データ・ソフト保守、HPCI特定研究有償課題（採択後利用可）等も可

## 必要性記載

**`C34` / `C63` / `C92` の3枚はすべて必ず埋める**。空欄のまま提出すると審査員が経費の妥当性を読み取れず、形式不備につながる。割合が90%超でなくても、それぞれ2〜4文の簡潔な必要性説明を記載する。

特に以下のケースは**詳しく**記載する：

1. 設備備品費が総額の90%超
2. 旅費が総額の90%超
3. 謝金が総額の90%超
4. 消耗品費・その他費目で**特に大きな割合**を占める経費がある

SPReAD では AWS 利用が「その他」に集中して大半を占めるケースが多い → **C92（その他費用の必要性）は特に丁寧に記載**する（AWS 選定理由、研究遂行上の不可欠性、4枚目との対応関係）。

必要性記入欄（merged C..N の左上、文字数制限なし）：
- `C34` 設備備品費・消耗品費の必要性 — 当該物品が研究遂行に不可欠な理由。**計上がない費目は「本研究では設備備品費・消耗品費の計上はない」と一行記載**して空欄を避ける。
- `C63` 謝金・旅費の必要性 — 謝金・旅費の使途と研究遂行上の必要性。**計上がない場合は「本研究では謝金・旅費の計上はない」と一行記載**する。
- `C92` その他費用の必要性 — AWS 計算資源費用・API 利用料の必要性、4枚目との対応。**SPReAD は実質ここが最重要**。

### 記載例（参考）

**C34 例（消耗品のみ計上の場合）**：
```
本研究では実験データ整理用の外付けストレージ（10TB）と関連参考書籍を消耗品費として計上する。
GPU 等の計算資源は AWS 上で確保するため、設備備品費は計上しない。
```

**C63 例（学会参加旅費のみの場合）**：
```
研究成果の中間報告と最新動向把握のため、人工知能学会全国大会（国内）への参加を予定し、
国内旅費を計上する。謝金は本研究では発生しない。
```

**C92 例（AWS 中心構成の場合）**：
```
本研究は LLM ファインチューニングに 80GB VRAM の GPU を要するため、所属機関の計算機では
不足する。AWS EC2（p4de.24xlarge 等）を利用することで必要な計算性能を確保する。
詳細な算定根拠は4枚目「計算資源費用」テーブルに記載。Bedrock API は文献メタデータ抽出に
利用予定で、4枚目「API費用」テーブルに対応。
```

## ヒアリングの進め方

1. 総額目処（10〜500 万円、典型 300〜500 万円）
2. 設備備品費を使うか（オンプレGPU購入、ワークステーション等）Yes/No
3. 消耗品費（書籍、ソフトウェア、周辺機器 等）
4. 謝金（被験者謝礼、外部講師 等、本事業は研究期間が短いので想定少なめ）
5. 旅費（学会参加、現地調査 等）
6. その他 — ここに AWS 費用総額（＝4枚目の合計）と API 費用を配分

AWS 中心案の典型配分例（500万円 ＝ 上限いっぱい）：
- 設備備品費：0 万円
- 消耗品費：20 万円（書籍・周辺機器）
- 謝金：0 万円
- 国内旅費：30 万円（学会参加1回）
- その他：450 万円（うち AWS 計算資源 400 万円、Bedrock API 50 万円）
- 合計：500 万円

## 書き込み時の注意

- 各費目の小計セル `J31`, `N31`, `E60`, `J60`, `N60`, `E89` は **テンプレ式のまま残す**。明細行の金額を入れれば自動で合計される（LibreOffice の recalc は通さないので、ユーザーが Excel で開いた瞬間に合計が表示される）。
- 1枚目総計 `D48` も `=SUM(E48:L48)` で各小計セルから自動連携。スクリプト側では一切触らない。
- 設備備品費は「単価 × 数量 = 金額」が自動計算なので、`G{row}` と `I{row}` だけ書き込み、`J{row}` は触らない。
- 20行を超える明細が必要な場合は、スクリプト側で警告を出すので、ユーザーに事項を集約してもらう（例：「ストレージ HDD 各種」のように1行で表現）。

## 単位換算の早見表

ユーザーとのやり取りは **万円** で統一、xlsx 様式への書き込みは **千円** で行う。

| 万円 | 千円（xlsx 書込値） |
|---:|---:|
| 10 万円（下限） | 100 |
| 50 万円 | 500 |
| 100 万円 | 1,000 |
| 300 万円 | 3,000 |
| 500 万円（上限） | 5,000 |

換算式：`千円 = 万円 × 10`
